# DockerForge v0.2.4

This patch release makes the DockerForge CLI safer when invoked without an explicit subcommand.

## Highlights

- `dockerforge` now prints help and does not generate files.
- `dockerforge .` is rejected instead of implicitly generating files.
- `dockerforge generate .` remains the explicit command for generating Docker assets in the current directory.
